## 推荐使用方式

### ⭐ 方案1：图片上传识别（最简单）

**适用场景**: 你已经验证这个方式工作正常！

**操作步骤**:
1. 在服务器上打开浏览器
2. 访问: http://localhost:8000
3. 点击顶部 **"图片上传"** 标签页
4. 点击"选择文件"上传照片
5. 自动识别并显示结果

**优点**:
- ✅ 简单直接
- ✅ 已验证工作正常
- ✅ 支持批量测试

## 其他使用方式

### 方案2：使用其他设备的摄像头

**适用场景**: 你有笔记本/手机等带摄像头的设备

**操作步骤**:

1. **确保设备在同一网络**
   - 设备和服务器必须在同一WiFi/局域网

2. **在有摄像头的设备上打开浏览器**
   - 可以是Windows笔记本、Mac、手机、平板等

3. **访问客户端页面**
   ```
   http://192.168.111.172:8000/static/local_camera_client.html
   ```

4. **使用摄像头**
   - 点击"启动本地摄像头"
   - 浏览器会请求摄像头权限，点击"允许"
   - 点击"拍照并识别"
   - 照片会自动上传到服务器识别
   - 查看识别结果

**优点**:
- ✅ 本地设备采集清晰
- ✅ 服务器算力识别
- ✅ 支持手机/平板

**注意事项**:
- 浏览器必须允许摄像头权限
- 非localhost访问可能需要HTTPS（有些浏览器限制）
- 确保网络连接稳定

---

### 方案3：连接USB摄像头（不推荐）

**适用场景**: 想在服务器本地使用摄像头

**需要**:
- 购买USB摄像头
- 物理连接到服务器
- 安装驱动（如需要）

**操作**:
```bash
# 连接USB摄像头后验证
ls /dev/video*

# 如果看到 /dev/video0，表示成功
```

**不推荐原因**:
- 需要额外硬件
- 服务器通常放在机房，不方便操作
- 方案1和方案2已经足够使用
